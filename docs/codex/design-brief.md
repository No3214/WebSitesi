# Kozbeyli Codex Design Brief

## Direction

The site is a premium, content-first hospitality experience. Keep the current light stone / olive / warm white direction as the default. Dark surfaces should be intentional accents, not page-wide defaults.

## Audience

- Couples looking for a quiet stone hotel experience.
- Families comparing rooms and nearby Foça access.
- Wedding, engagement, and event planners checking venue credibility.
- International guests who need complete English pages and booking clarity.

## Signature Element

Use real Kozbeyli media as the signature element: opening video, room photography, food preparation footage, and wedding/event documents. Do not add generated or uncertain hotel imagery for rooms, food, weddings, team, or property proof.

## UI Rules

- Use readable editorial hierarchy: serif headings, restrained body copy, clear CTA labels.
- Keep mobile bottom actions visible and simple: call, location, booking.
- Booking CTAs must open the HMS booking engine directly, not a cramped embedded widget.
- Preserve content truthfulness over decorative claims: no unverified review scores, historical claims, cancellation guarantees, or fixed wedding prices.
- For new interface primitives, prefer existing local components first; use shadcn only when it improves semantics or accessibility.

## Motion Rule

Animation must support hospitality pacing. Use subtle transitions and scroll reveals; avoid gimmicks, fake loading theatrics, and heavy effects that delay booking or media playback.
