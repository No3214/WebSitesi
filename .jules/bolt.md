## 2024-05-18 - Scroll Event Optimization in React Components
**Learning:** React components that use high-frequency event listeners like `window.addEventListener("scroll", ...)` can cause excessive internal overhead and React re-renders. The `Parallax` and `SiteHeader` components directly attach to the scroll event without debouncing or tracking the state efficiently.
**Action:** When tracking scroll state (like `scrolled > 24`), use a local variable within the closure to track the state, and only call the React state dispatcher when the value actually transitions. For example: `const isScrolled = window.scrollY > 24; if (isScrolled !== scrolled) setScrolled(isScrolled);` - but `scrolled` in the dependency array would cause re-attaching. The best way is `let currentScrolled = window.scrollY > 24; setScrolled(currentScrolled); const onScroll = () => { const newScrolled = window.scrollY > 24; if (newScrolled !== currentScrolled) { currentScrolled = newScrolled; setScrolled(currentScrolled); } }`.
