## 2026-06-10 - [Insecure Direct Object Reference (IDOR) / Input Validation in Checkout API]
**Vulnerability:** The checkout API endpoint (`src/app/api/checkout/route.ts`) relied on unvalidated client-side input for room pricing and metadata. Specifically, it accepted `data.roomSlug` and `data.roomTitle` directly from the request payload to determine the price rate and to construct the reservation message saved to the database. This allowed malicious users to manipulate the payload to book an expensive room at a lower rate by passing a mismatched title and slug.
**Learning:** Client-side payloads, even when parsed by a schema like Zod, cannot be trusted for critical business logic like pricing or product association unless validated against a server-side source of truth.
**Prevention:** Always use the client-provided identifier (e.g., `roomSlug`) to look up the authoritative entity from the server's database or static data source (e.g., `src/data/rooms.ts`). Use the server-side entity for all subsequent calculations, processing, and database persistence.
